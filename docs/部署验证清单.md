# NexusOrbital认证系统 - 部署验证清单

本文档提供了在Vercel部署完成后验证认证系统各功能是否正常工作的详细清单。请逐项检查以确保系统正常运行。

## 基础检查

### 1. 网站可访问性

- [ ] 访问部署URL（例如：`https://nexusorbital-website.vercel.app`）
- [ ] 确认网站主页正常加载（200状态码）
- [ ] 确认页面样式和资源（CSS、JS、图片）正常加载
- [ ] 检查移动端和桌面端页面显示是否正常

### 2. 健康检查API

- [ ] 访问`/api/health`端点
- [ ] 确认返回200状态码
- [ ] 确认JSON响应包含`status: "ok"`
- [ ] 确认响应包含当前时间和版本信息

## 认证功能测试

### 3. 用户注册功能

- [ ] 访问`/auth-test`或其他包含注册功能的页面
- [ ] 使用测试邮箱填写并提交注册表单
- [ ] 确认收到成功响应（201状态码）
- [ ] 确认响应包含JWT令牌和用户信息
- [ ] 确认注册的用户名、邮箱与提交的数据一致

### 4. 用户登录功能

- [ ] 在登录页面输入刚注册的凭据
- [ ] 提交登录表单
- [ ] 确认收到成功响应（200状态码）
- [ ] 确认响应包含JWT令牌和用户信息
- [ ] 确认登录后能正确识别用户身份

### 5. 短信验证码功能

- [ ] 访问验证码测试页面
- [ ] 输入国家代码和手机号码
- [ ] 点击发送验证码按钮
- [ ] 确认收到成功响应（200状态码）
- [ ] 验证开发环境中是否显示了验证码（仅开发环境适用）

### 6. 令牌验证功能

- [ ] 使用登录或注册获取的JWT令牌
- [ ] 发送令牌验证请求
- [ ] 确认收到成功响应（200状态码）
- [ ] 确认响应包含正确的用户信息
- [ ] 确认令牌未过期

## 安全性验证

### 7. 未授权访问测试

- [ ] 尝试不带令牌访问需要授权的端点
- [ ] 确认返回401或403状态码
- [ ] 确认错误消息提示需要授权

### 8. 令牌过期测试（可选）

- [ ] 修改系统时间或等待令牌超时
- [ ] 使用过期令牌尝试访问需要授权的端点
- [ ] 确认返回403状态码和过期提示

### 9. 错误处理测试

- [ ] 输入无效的邮箱格式进行注册
- [ ] 确认返回400状态码并有明确错误提示
- [ ] 使用不存在的用户尝试登录
- [ ] 确认返回适当的错误状态和消息

## 集成验证

### 10. 前端集成

- [ ] 在主网站中配置API基础URL为部署地址
- [ ] 测试注册、登录、验证等功能
- [ ] 确认前端正确处理API响应和错误
- [ ] 确认JWT令牌正确存储在localStorage或cookies中

### 11. 登录状态持久化

- [ ] 登录后刷新页面
- [ ] 确认用户仍保持登录状态
- [ ] 关闭浏览器并重新打开网站
- [ ] 确认根据设计用户保持登录或需要重新登录

## 性能验证

### 12. 响应时间

- [ ] 测量API响应时间是否在可接受范围（通常<500ms）
- [ ] 检查是否有明显的延迟或超时
- [ ] 测试在多个地理位置的访问速度（可选）

### 13. 并发请求

- [ ] 使用工具模拟多个并发请求（可选）
- [ ] 确认系统能够正常处理并发负载
- [ ] 检查是否有请求失败或响应变慢

## 环境变量验证

### 14. 机密变量检查

- [ ] 在Vercel控制台确认`JWT_SECRET`已正确设置
- [ ] 确认生产环境中不显示敏感信息（如验证码）
- [ ] 测试JWT令牌是否使用了正确的密钥签名
